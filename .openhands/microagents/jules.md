# Jules 連携 MicroAgent

## 役割
Jules は Google の Gemini 2.5 Pro を基盤とした強力な実装エージェントである。OpenHands エージェント（現場監督）が、複雑な実装が必要なタスクを識別した際、Jules に作業を委任することができる。

## コマンド
`jules task "タスクの詳細指示"` を実行して作業を開始させる。

## 作業ディレクトリ
コマンド実行前に、必ず対象の Git サブモジュールディレクトリに移動すること。

## 非同期ワークフローの理解
1. `jules task` 実行後、Jules は独自の VM 上で作業を開始し、終了すると Pull Request (PR) を作成する。
2. OpenHands エージェントは、コマンドの戻り値（タスク ID 等）を記録し、一旦他のタスク（進捗更新など）を進めて良い。
3. `jules status <task_id>` 等で進捗を定期確認するか、GitHub 上の新規 PR の出現を監視する。

## 委任の判断基準
- 大規模な新規機能開発。
- 複雑なリファクタリング。
- 多数のファイルに跨る修正。
