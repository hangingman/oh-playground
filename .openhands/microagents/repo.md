# リポジトリ MicroAgent

本プロジェクトは「AIによる完全自律開発の実験場」です。本リポジトリ (`oh-playground`) は環境定義のためのメタリポジトリであり、実際のプロダクト開発は OpenHands によってワークスペース内にクローンされた各プロジェクトディレクトリ内で行われます。

## プロジェクトディレクトリを作業対象とする開発フロー

1. **初期化と環境認識**:
   - OpenHandsの初回起動時に選択されたリポジトリは、通常ワークスペース直下（`/opt/openhands/workspace/<repo-name>`）にクローンされます。
   - 開発を開始する前に、対象リポジトリのディレクトリが存在し、必要なファイルが揃っていることを確認してください。
2. **作業境界の認識**:
   - あなたの主要な作業範囲は、クローンされたプロジェクトディレクトリ内です。
   - `oh-playground` 側の環境設定（メタ設定）と、開発対象プロジェクトのコードを明確に区別して扱ってください。
3. **Git 操作**:
   - プロダクトに関するコミットやプッシュは、原則としてプロジェクトディレクトリ内で行ってください。
   - OpenHandsに設定された Git 認証情報を利用し、自律的に変更を反映させることが可能です。
4. **Memory Bank の更新**:
   - プロジェクト固有の変更や意思決定は、`memory-bank/` 内のドキュメントに記録してください。常に `activeContext.md` を参照し、現在どのプロジェクトが作業対象であるかを把握してください。

## 監督-施工 (Supervisor-Worker) ワークフロー

本プロジェクトでは、OpenHands が「現場監督」、Jules が「熟練工」として機能する分業体制を採用しています。

- **OpenHands (現場監督)**:
  - プロジェクト全体の進捗管理、設計書の作成、タスクの切り出し。
  - Jules への作業発注と成果物（PR）のレビュー・統合。
- **Jules (熟練工)**:
  - 特定の重厚なコーディングタスクの実装。
  - 複雑なリファクタリングや新規機能の実装。

**Jules への委任指針**:
複雑なリファクタリング、大規模な新規機能開発、あるいは広範囲に及ぶ修正が必要な場合は、自らコードを書くのではなく、Jules CLI (`jules task`) を使用して Jules に作業を委任することを強く推奨します。

## コーディング規約

- **Python 3.10以上**を使用してください。
- **PEP8**に準拠したコーディングを心がけてください。
- 全ての新規機能に対して、適切な**ユニットテスト**を作成してください。
- コードを提出（Submit）する前に、エージェント自身による**セルフレビュー**を必ず実施してください。

## 自律化のゴール

- **テストカバレッジ 100%** を維持すること。
- ドキュメントを自動生成し、常に最新の状態に保つこと。
- バグが発見された際、自動的に修正案を提示・適用する**自己修復（Self-healing）**機能を強化すること。
- 人間の介入を最小限に抑え、高品質なコードを自律的に生成・維持し続けること。
