# リポジトリ MicroAgent

本プロジェクトは「AIによる完全自律開発の実験場」です。本リポジトリ (`oh-playground`) は環境定義のためのメタリポジトリであり、実際のプロダクト開発は Git submodule 内で行われます。

## サブモジュールを主作業対象とする開発フロー

1. **初期化 (ステップ 0)**:
   - 開発を開始する前に、必ず `git submodule update --init --recursive` を実行し、開発対象のサブモジュールが最新の状態であることを確認してください。
2. **作業境界の認識**:
   - あなたの主要な作業範囲は `./<submodule-name>/` ディレクトリ内です。
   - プレイグラウンド側の設定（メタ設定）と、プロダクト側のコード（サブモジュール内）を混同しないでください。
3. **Git 操作**:
   - プロダクトに関するコミットやプッシュは、必ず該当するサブモジュールディレクトリ内で行ってください。
4. **Memory Bank の更新**:
   - プロダクト固有の変更は、サブモジュール内の文脈として扱ってください。`memory-bank/activeContext.md` 等を確認し、現在どのサブモジュールが作業対象であるかを常に把握してください。

## コーディング規約

- **Python 3.10以上**を使用してください。
- **PEP8**に準拠したコーディングを心がけてください。
- 全ての新規機能に対して、適切な**ユニットテスト**を作成してください。
- コードを提出（Submit）する前に、エージェント自身による**セルフレビュー**を必ず実施してください。

## 自律化のゴール

- **テストカバレッジ 100%** を維持すること。
- ドキュメントを自動生成し、常に最新の状態に保つこと。
- バグが発見された際、自動的に修正案を提示・適用する**自己修復（Self-healing）**機能を強化すること。
- 人間の介入を最小限に抑え、高品質なコードを自律的に生成・維持し続けること。
