# システム構成 (System Patterns)

## アーキテクチャ概要
本プロジェクトは OpenHands をベースとした自律開発環境であり、**メタリポジトリ構造**を採用しています。
`oh-playground` 自体は開発環境とルールを定義し、実際のプロダクトコードは Git サブモジュールとして管理されます。

## 主要なコンポーネント
- **OpenHands**: 実行エンジン。自律的なエージェントとして動作します。
- **MicroAgents**: 特定のタスクや規約に基づいた AI の思考をガイドするコンポーネント。`.openhands/microagents/repo.md` にサブモジュール指向の開発ルールが定義されています。
- **Memory Bank**: プロジェクトの長期・短期記憶を保持する Markdown ドキュメント群。
- **Git Submodules**: 実際の開発対象となるプロダクト。`./<submodule-name>/` 配下に展開されます。
- **Jules CLI**: Google の AI エージェント「Jules」を操作するための CLI ツール。重厚な実装タスクを Jules に委任するために使用されます。

## 重要な設計判断
- **サブモジュール分離**: プレイグラウンドの設定とプロダクトのコードを分離することで、環境の汚染を防ぎ、複数のプロジェクトを切り替えて実験することを可能にします。
- **Memory Bank 優先**: Cline などの AI アシスタントは、作業前に必ず Memory Bank を参照し、作業後に更新することで、文脈の喪失を防ぎます。プロダクト固有の文脈も Memory Bank で管理します。
- **自律的 QA**: サブモジュール内での実装と同時にテストコードを作成し、自律的に検証を行うパターンを採用します。
- **監督-施工ワークフロー**: OpenHands（監督）が全体の進捗と設計を管理し、Jules（施工）が具体的なコーディングを行うマルチエージェント・ワークフロー。これにより、複雑なタスクの並行処理と高品質な実装の両立を目指します。
